## Why

同一體系內存在姓名完全相同的學員（例如兩位「王小明」），日常查詢時難以辨識，容易找錯人、改錯資料。目前系統以姓名搜尋只會列出符合字串的結果，無法一眼看出「哪些名字在體系內是重複的」，也沒有把同名者集中檢視的方式。

## What Changes

- 學員管理頁的快捷視圖新增一個 **「同名學員」**（`duplicate_name`）：一鍵列出**目前體系內姓名完全相同**（去除前後空白後字串相等）且**出現 2 人以上**的所有學員。
- 結果依姓名分群排序，讓同名的人相鄰顯示，便於逐一比對辨識（而非散落各處）。
- 沿用既有快捷視圖的互動：點選套用、再點取消，與其他快捷視圖互斥。
- 沿用既有體系隔離：只在登入者的有效體系內判定重複，不跨體系比對。
- 匯出沿用「匯出＝畫面所見」，套用此視圖後匯出即為同名名單。

## Capabilities

### Modified Capabilities
- `smart-filters`: 情境快捷視圖新增「同名學員」視圖；此視圖的判定需先彙整全體系姓名出現次數（跨列統計），與既有逐列判定的視圖不同，需在規格中明確其語意與排序。

## Impact

- 型別：`lib/db/types.ts`（`StudentView` 新增 `'duplicate_name'`）。
- 篩選核心：`lib/db/supabaseRepository.ts`——既有快捷視圖走「全量載入 + JS 過濾 + slice 分頁」路徑；本視圖需在全量載入後、過濾前**先建立姓名次數表**，再據此過濾並依姓名分群排序。
- UI：`components/StudentGrid/FilterBar.tsx`（新增快捷視圖按鈕）。
- 匯出：`app/api/export/route.ts`（快捷視圖條件需同步支援，維持「匯出＝畫面所見」）。
- 假資料來源：`lib/db/mockRepository.ts`（如有支援 view 需同步）。
- 不需 DB migration（純查詢/前端邏輯，無 schema 變更）。
